# /improve-ux

UI/UX 개선 사항을 분석하고 제안합니다.

## Usage
```
/improve-ux [options]
```

## Options
- `--platform <type>` - 대상 플랫폼: `web`, `mobile`, `all` (기본값)
- `--persona <type>` - 사용자 페르소나: `new-user`, `casual`, `power-user`, `all`
- `--aspect <area>` - 특정 UX 영역: `onboarding`, `navigation`, `forms`, `data-viz`, `accessibility`, `performance`
- `--priority` - 구현 우선순위별 정렬

## Implementation

프로젝트의 현재 UI/UX를 분석하여 다음 항목들을 평가하고 개선 방안을 제시합니다:

1. **사용성 휴리스틱 평가**
   - 시스템 상태의 가시성
   - 시스템과 현실 세계의 일치
   - 사용자 제어와 자유
   - 일관성과 표준
   - 오류 방지
   - 인식보다는 회상
   - 유연성과 효율성
   - 미적이고 미니멀한 디자인
   - 오류 복구
   - 도움말과 문서

2. **사용자 여정 매핑**
   - 진입점 분석
   - 핵심 경로 최적화
   - 문제점 파악
   - 이탈 지점 개선
   - 전환 기회 포착

3. **인터랙션 디자인 패턴**
   - 마이크로인터랙션
   - 피드백 메커니즘
   - 로딩 상태
   - 빈 상태
   - 오류 상태

4. **시각적 디자인 분석**
   - 타이포그래피 계층
   - 색상 심리학
   - 간격과 레이아웃
   - 시각적 흐름
   - 브랜드 일관성

5. **모바일 최적화**
   - 터치 타겟 크기
   - 제스처 인터랙션
   - 반응형 디자인
   - 성능 최적화

6. **접근성 개선**
   - WCAG 준수
   - 키보드 네비게이션
   - 스크린 리더 지원
   - 색상 대비

결과물로 다음을 제공합니다:
- 현재 UX 점수 (100점 만점)
- 즉시 적용 가능한 개선사항 (Quick Wins)
- 중요도 높은 개선사항
- 구체적인 구현 예시 코드
- 예상 사용자 만족도 증가율
- 구현 로드맵
